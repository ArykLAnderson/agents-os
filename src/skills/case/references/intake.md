# Intake

Intake already-understood sources into cohesive reusable knowledge.

1. Confirm the source purpose and intended reusable meaning are clear.
2. Select an existing Case or create one whose subject, authority, lifecycle, retrieval context, and distribution boundary fit.
3. Register each independently identifiable source and its examination objective.
4. Extract purpose-relevant semantic units with initial classifications, support, authority, inference disclosure, and relationships.
5. Run the full validation checklist before completing the batch. Repair only derived or mechanical defects automatically; leave semantic defects explicit.

Perform only narrow lookups needed for identity, provenance, authority, or obvious overlap. Use Frame when purpose, interpretation, trade-offs, or consequential decisions are unclear.

A projection Case may synthesize explicitly selected Case entries. Record the exact `case:<id>#CK-<id>` inputs in a `## Projection Basis` section and cite the relevant basis entries from every derived entry. It remains an ordinary Case with its own purpose, scope, authority, support, and classifications.
