# Visual Anchor

Use one lightweight anchor for each meaning-bearing visual retained during shaping. It is part of the semantic artifact, not a rendering specification.

A visual anchor is a semantic brief, not an artifact locator. Identify it with a stable ID such as `VA-001` and give the shaped artifact a separate semantic locator for the beat where it belongs.

## Required Fields

- **Reader question:** the one question the visual answers.
- **Placement:** the explanatory beat it follows or precedes.
- **Takeaway:** the one sentence a reader should retain.
- **Prerequisites:** concepts and context established before the visual.
- **Must show:** reader-relevant relationships, comparisons, quantities, or hierarchy.
- **Keep in prose:** material meaning that need not be spatially encoded.
- **Omit:** implementation, trace, and validation detail beyond the visual's cognitive budget.
- **Forbidden implications:** false sequence, causality, completeness, authority, precision, priority, or mandatory action.
- **Selected semantic form:** comparison table, cards, boundary model, lineage, chart, diagram, or another named form. A target-specific realization is not selected here.

## Completion Check

Create and retain a visual anchor only when the planned visual is expected to reduce reader effort relative to surrounding prose alone. Split or distribute the anchor when it would answer more than one reader question. Close the opportunity as `converted-to-prose` or `declined` when its planned caption or textual equivalent would need to correct the visual's likely dominant reading.
- **ID and source opportunity:** stable `VA-###` and originating `VO-###`.
- **Artifact revision and semantic locator:** the shaped revision and visible or hidden beat locator.
