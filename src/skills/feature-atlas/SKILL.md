# Feature Atlas Representations

1. Read [the canonical Issue representations](references/issue-representations.md) for identity, ownership, current-state bodies, and historical comments.
2. When the configured canonical tracker is private GitHub Issues, also follow [the configured private GitHub instructions](references/configured-private-github.md).
3. Keep tracker mechanics separate from Feature Atlas semantics. A tracker locator or native parent relationship aids navigation but never defines domain identity or ownership.

This skill defines representations only. It does not define Feature shaping, specification, decomposition, publication acceptance, reconciliation, migration, or delivery workflows.
