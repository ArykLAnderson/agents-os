# Mental Model Strategy

Use for explanation documents whose reader must understand concepts and relationships before acting on a conclusion.

## Reader Journey

1. Name the practical problem and bounded reader action.
2. Define terms in dependency order, beginning with the durable object or record.
3. Explain lifecycle and relationships with a textual path before or alongside a visual.
4. Use one representative example without claiming it is exhaustive.
5. State the safety boundary, proof boundary, and what the model cannot establish.
6. End with the reader action or a handoff to a review briefing.

## Anchors And Fit

Use stable anchors for `problem`, `vocabulary`, `lifecycle`, `relationships`, `example`, `boundaries`, and `next-action`. Define a term once before using its shorthand. Label examples and diagrams as bounded representations.

This strategy does not fit when the reader already understands the system and only needs a narrowly scoped approval or review ask; use `review-briefing` then. It must not replace a Case decision, source trace, or approval record.
