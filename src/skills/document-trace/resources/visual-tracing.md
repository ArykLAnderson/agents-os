# Visual Tracing

Trace a table, diagram, generated image, or other visual as a semantic unit when it carries information required for the reader action.

For each material visual or table, record its visual anchor, takeaway, supported reader-facing relationships or comparisons, and any boundary needed to avoid a false implication. Distinguish meaning that must appear visually from meaning intentionally preserved in adjacent prose. Trace the caption and visual independently when either makes a different claim.

Trace coverage constrains what a visual may claim; it does not require the visual to display every supporting entry, internal operation, artifact, edge, or validation identifier. Treat node IDs, source locators, exhaustive operation inventories, and trace accounting as internal unless the reader action genuinely depends on them.

Block the artifact when a visual implies approval, sequence, causality, completeness, priority, quantitative precision, or an unqualified recommendation not supported by its Case entries. If a visual only decorates already-supported prose and carries no new claim, record it as non-material rather than inventing trace detail.
