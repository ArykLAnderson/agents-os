# Coverage Check

Before review, compare the shaped artifact, composition manifest, and trace sidecar.

- Every material recommendation, conclusion, limitation, contradiction disposition, decision request, and evidence-table conclusion has one trace unit.
- Every selected Case entry is visible, omitted, or deferred in the trace accounting.
- Every omitted or deferred material entry has a reason and does not silently change reader action.
- Every trace support reference is fully qualified and belongs to a pinned snapshot.
- Every material table, figure, or callout has a traceable semantic summary; a table needs one unit per material row when one unit cannot honestly cover each row's distinct claim.
- The artifact's visible anchors match the locator named by each visible trace unit.

Report each failure with the artifact locator, missing or conflicting Case reference, severity, and disposition. These are trace blockers: unsupported decision or evidence; stale support; omitted material selected entry without accounting; material table or visual claim without a unit; or a Case-backed reader action unsupported by the Case. An externally supplied review instruction is not Case-backed: label it as external context and do not treat it as supported, blocked, or publishable. Do not infer support from the artifact title, a source filename, or the mutable Case ledger, and do not demand trace coverage for connective prose that makes no independent semantic assertion.
