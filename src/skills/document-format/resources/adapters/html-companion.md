# HTML Companion Adapter

Use a local HTML representation when a shaped artifact benefits from navigation or a visual aid. It may be the final local deliverable and remains traceable to the semantic source.

**Representation conditions:** local HTML can satisfy final-representation review through local inspection. This adapter has no remote destination or native stage/release behavior; offer a manual handoff or a separately selected destination adapter for publication.

## Required Output

- Use semantic landmarks, a document language, one visible `h1`, ordered headings, descriptive page title, and keyboard-reachable navigation.
- Keep the traced recommendation, evidence status, limitation, and reader action as ordinary selectable text in reading order.
- Resolve every local image, stylesheet, and document link. Give informative images an `alt` equivalent and decorative images empty `alt` text.
- Local assets may support the rendered package, but reader-facing evidence links must be resolvable by the intended audience. Do not expose author-machine paths, Case paths, or private trace locators as citations or source links.
- Include a textual equivalent for every semantic visual adjacent to the visual or in the same section.
- Support ordinary and narrow layouts without clipping, horizontal table loss, or navigation that hides the current conclusion.

## Local Evidence

Inspect the HTML at a normal desktop width and a narrow mobile width. Record link and asset resolution, keyboard navigation order, text alternatives, searchable critical phrases, and any residual rendering limitation.
