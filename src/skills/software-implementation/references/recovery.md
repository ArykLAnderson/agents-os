# Recovery

## Ordinary Repair

Give bounded validator findings to the original Coding Worker when its session and exclusive worktree ownership remain available. Otherwise use a fresh worker with the complete Task Contract, safe source identity, prior evidence, and findings. Continuation is an optimization; compact handoff is sufficient.

Add design-compatible remediation tasks autonomously only when they preserve accepted behavior, Atlas identities/ownership, dependencies, convergence, proof ordering, limitations, architecture, scope, and Effect Bindings. Put them through the same admitted worker, focused-validation, integration, convergence, and imported proof gates.

## Repeated No Progress

Track only compact attempt summaries and the current blocker in the execution map.

1. After repeated evidence of the same no-progress approach, require a materially different worker approach.
2. If an independent attempt reaches the same blocker, delegate one bounded read-only diagnosis focused on the discriminating technical question.
3. Convert a diagnosis into a bounded writer task only when it is design-compatible and authorized.
4. Return a concrete material contradiction to the governing human/Frame/Blueprint/Atlas authority. Do not reopen design for difficulty, preference, or a speculative alternative.

A bounded diagnosis investigates; it neither edits nor certifies.

## Interrupted Or Unknown Writers

Ask the Harness Adapter for truthful inspect/resume/cancel capability. Before reusing source state, Workspace Operator must confirm writer cessation and exclusive ownership. If status is unknown, quarantine the old worktree and create a fresh one from the last safe integrated or committed baseline.

Never let an uncertain session and replacement writer share a worktree.

## Coordinator Resumption

Open the stable execution-map locator, then reconcile:

- the persisted Atlas Delivery Binding against a fresh resume Currentness Check through Feature Atlas domain operations;
- task/gate states against the admitted graph, Git branches, and worktrees;
- role/session notes against adapter inspection;
- evidence locators, qualifications, and invalidators against candidate source/external state;
- effects against their bound external systems and cleanup records; and
- PR state against the bound provider/repository.

Correct stale map text rather than treating it as authority. Do not use provider CLI/path discovery as a substitute for the configured Atlas domain adapter. Concurrent coordinator ownership is unsupported.

## Atlas Successor, Invalidator, Or Integrity Stop

Do not follow a newer Decision. If the bound Decision is no longer current, publication conflicts, or a binding becomes unverifiable, stop affected dispatch/effects/results and preserve coherent source/effect state. Resume only from a newly admitted current handoff and explicit execution disposition that says what prior work/evidence may be salvaged.

A triggered/unknown invalidator or typed limitation may yield `exact_admitted_limitation` only when the admitted handoff already defines that exact state, affected claims/work, forbidden claims, allowed unaffected boundary, and owner. Otherwise stop. The coordinator cannot infer semantic equivalence, rebind local labels, or promote old evidence.

## Authority Stops

Technical blockage cannot broaden accepted design, Atlas meaning, external effects, spend, provider identity, PR permission, or landing authority. Preserve a coherent source disposition and return the smallest decision/evidence packet to the authority that owns the contradiction.
