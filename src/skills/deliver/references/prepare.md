# Prepare Bounded Work

Select only Work Items whose accepted prerequisites are currently established by source-system evidence. Ordering preference alone does not block or enable work.

For each Hand assignment provide:

- exact Route/Leg/Work Item identities and accepted revisions;
- one bounded responsibility and implementation context;
- Blueprint Contracts realized or preserved;
- immediate consumer and integration seam;
- checkable seam-local completion;
- independent Clerk method/checker expectation;
- allowed files/systems and source/worktree boundary;
- effect authorization, prohibited effects, and escalation stops;
- temporary-resource owner, location, authority class, retention/expiry, and cleanup evidence;
- known Findings, limitations, and required source locators.

Do not ask a Hand to resolve behavior, architecture, Route strategy, dependency meaning, or human authority. Return those uncertainties to Frame, Blueprint, Route, or the Architect before execution.

Use one writer for one source boundary. Parallelize only genuinely independent Work Items or use isolated worktrees. A Hand is already the worker and does not recursively delegate unless explicitly configured as a bounded coordinator.
