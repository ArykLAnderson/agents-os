# Materiality Triage

Classify a suspected semantic failure against accepted behavior, Blueprint, Route, Leg, dependency, consumer, evidence, and cleanup assumptions.

## Clearly local

Repair only when all accepted assumptions remain unchanged and the defect is bounded implementation, integration, workmanship, or missing evidence. Record the source/evidence delta and repeat independent verification.

## Clearly material

If correction requires changing accepted behavior; Blueprint responsibility/Contract; Route strategy; Leg boundary, order, acceptance, or dependency; migration/compatibility; evidence strategy; cleanup; transition assumption; or downstream expectation:

1. stop affected execution;
2. prevent unsupported completion claims;
3. preserve source/Runtime evidence and temporary-effect obligations;
4. close Deliver as `route-invalidated`;
5. return evidence to Route and, when needed, Blueprint/Frame;
6. require a fresh human-accepted Route candidate and fresh Deliver identity.

Existing implementation is evidence or potential salvage only and must re-earn inclusion.

## Genuinely unclear

Suspend only the affected frontier for one bounded diagnosis:

- state one discriminating question and small budget;
- permit no production repair or Blueprint/Route mutation;
- preserve authorizations and cleanup obligations without expanding them;
- require independent Clerk boundary review;
- return evidence to the Architect for local/material classification.

A local answer resumes this Deliver. Confirmed materiality terminates it. Diagnosis is not in-place replanning.
