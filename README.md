# agents-os

Portable global Agent OS + Pi setup.

This repo is the canonical home for my global agent configuration, Pi harness config, and custom Pi plugins. It is designed to be cloned to `~/.agents-os` and then linked into Pi via `~/.pi`.

## Layout

```text
~/.agents-os/
  src/                    # canonical Agent OS source
  config/                 # Agent OS adapter/model config
  adapters/               # generated adapter output
  harness_config/
    pi/
      agent/
        settings.json
        keybindings.json
        themes/
        extensions/
  plugins/
    pi-keysmith/
```

## What is included

- Global Agent OS source (`src/`)
- Agent OS config (`config/`)
- Pi global config (`harness_config/pi/agent/...`)
- Custom Pi extensions under `harness_config/pi/agent/extensions/`
- Custom Pi plugin source under `plugins/`

## What is intentionally excluded

This repo ignores personal/dev artifacts and runtime state, including:

- `docs/`
- `archive/`
- Pi auth/session/history/intercom/runtime files
- agent memory snapshots under `src/memory/agents/`
- plugin build artifacts and `node_modules/`

See `.gitignore` for the exact rules.

## Install

Clone to the canonical location:

```bash
git clone git@github.com:<your-username>/agents-os.git ~/.agents-os
```

Then link Pi to the canonical config root:

```bash
mv ~/.pi ~/.pi.backup 2>/dev/null || true
ln -s ~/.agents-os/harness_config/pi ~/.pi
```

## Pi settings assumptions

`harness_config/pi/agent/settings.json` expects:

- generated Agent OS Pi skills at `~/.agents-os/adapters/pi/generated/skills`
- generated Agent OS Pi commands at `~/.agents-os/adapters/pi/generated/commands`
- local pi-keysmith plugin at `~/.agents-os/plugins/pi-keysmith/packages/pi-keysmith`

## Plugin bootstrap

For local plugins, install dependencies/build as needed.

Example for `pi-keysmith`:

```bash
cd ~/.agents-os/plugins/pi-keysmith
npm install
npm run build
```

## Updating generated adapters

`src/` is the source of truth. `adapters/*/generated/` can be regenerated by the Agent OS sync workflow.

## Publish checklist

Before pushing:

- Review `git status --ignored` and confirm only intended files are tracked.
- Confirm `docs/`, runtime state, auth files, and local-only personal workflow files remain ignored.
- Grep for accidental private references:

```bash
rg -n '/Users/|api[_-]?key|secret|password' .
```

- If plugin packages are used locally, install/build them after clone.
- Reconfirm `~/.pi` points to `~/.agents-os/harness_config/pi`.

## Notes

- `~/.pi` is treated as a compatibility symlink, not the source of truth.
- This repo is meant to stay portable across personal/work machines.
